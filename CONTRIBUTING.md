# Contributing to Hyper-V Lab Environment

Thank you for your interest in contributing to the Hyper-V Lab Environment project! We welcome contributions, including bug reports, feature requests, documentation improvements, and code enhancements.

## How to Contribute

### Reporting Issues

If you discover a bug or have a question, please:

1. **Check existing issues** - Search the [issue tracker](https://github.com/MSDLA/hyper-v-lab-environment-/issues) to see if your issue has already been reported
2. **Create a new issue** - If not, [open a new issue](https://github.com/MSDLA/hyper-v-lab-environment-/issues/new) with:
   - Clear, descriptive title
   - Detailed description of the problem
   - Steps to reproduce (if applicable)
   - Expected behavior vs actual behavior
   - Environment details (OS, Hyper-V version, etc.)
   - Relevant logs or error messages

### Suggesting Enhancements

We love feature ideas! To suggest an enhancement:

1. **Check existing discussions** - Review open issues and discussions first
2. **Open an issue** with the `enhancement` tag
3. **Provide context** - Explain why this feature would be useful
4. **Include examples** - Show how it would work

### Submitting Changes

To contribute code or documentation:

1. **Fork the repository** - Click the "Fork" button on GitHub
2. **Create a feature branch** - `git checkout -b feature/your-feature-name`
3. **Make your changes** - Keep commits clear and focused
4. **Test thoroughly** - Ensure your changes work as expected
5. **Commit with clear messages** - Write descriptive commit messages
6. **Push to your fork** - `git push origin feature/your-feature-name`
7. **Submit a Pull Request** - Include:
   - Clear description of changes
   - Reference to related issues (if any)
   - Screenshots or examples (if applicable)
   - Testing details

### Documentation Contributions

Documentation improvements are always welcome! You can:

- Fix typos and grammar
- Clarify confusing sections
- Add examples and use cases
- Improve setup guides
- Contribute new guides or tutorials

## Code Style Guidelines

- Use clear, descriptive names for variables and functions
- Add comments for complex logic
- Follow PowerShell conventions (PascalCase for functions, lowercase for variables)
- Include error handling
- Test scripts before submitting

## Documentation Style

- Use clear, concise language
- Include code examples where applicable
- Add table of contents for long documents
- Use consistent formatting and headings
- Include screenshots for visual guides (when helpful)

## Pull Request Process

1. Update documentation to reflect any changes
2. Add comments explaining your code
3. Ensure scripts have proper error handling
4. Test your changes thoroughly
5. Keep pull requests focused - one feature/fix per PR
6. Link to related issues in the PR description

## Conduct

Please be respectful and constructive in all interactions. We're building this community together and appreciate every contribution, regardless of its size.

## Questions?

- Open a discussion in the [Discussions](https://github.com/MSDLA/hyper-v-lab-environment-/discussions) tab
- Check the [Troubleshooting Guide](./docs/troubleshooting.md)
- Review existing documentation

## Recognition

Contributors will be recognized in:
- Project README (if desired)
- Commit history
- Release notes

Thank you for making this project better! 🙏
